## 3. 核心模块详细功能

### 3.1 IoT 感知层

```mermaid
flowchart TB
    subgraph 传感器["传感器集群"]
        S1["人流计数器<br/>红外/TOF"]
        S2["气体传感器<br/>NH₃/H₂S"]
        S3["耗材传感器<br/>重量/红外"]
        S4["水浸传感器"]
        S5["门磁传感器<br/>坑位占用"]
        S6["温湿度传感器"]
    end

    subgraph 边缘["边缘网关"]
        GW["数据汇聚<br/>协议转换<br/>本地预处理<br/>断网缓存"]
    end

    subgraph 云端["云平台"]
        MQ["消息队列<br/>MQTT Broker"]
        DP["数据处理<br/>清洗/聚合"]
        DB["时序数据库"]
        AL["告警引擎"]
    end

    S1 & S2 & S3 & S4 & S5 & S6 --> GW
    GW -->|MQTT/NB-IoT| MQ
    MQ --> DP --> DB
    DP --> AL

    style 传感器 fill:#e8f5e9,stroke:#2e7d32
    style 边缘 fill:#fff3e0,stroke:#e65100
    style 云端 fill:#e3f2fd,stroke:#1565c0
```

| 功能 | 描述 | 采集频率 | 告警规则 |
|------|------|---------|---------|
| 人流计数 | 统计进出人次，计算累计使用量 | 实时（事件驱动） | 累计 N 人次后触发清洁任务 |
| 空气质量 | 监测 NH₃、H₂S、VOC 浓度 | 30 秒 | 浓度超阈值连续 2 次告警 |
| 耗材余量 | 纸巾卷余量、皂液余量 | 5 分钟 | 余量 ≤ 20% 提醒、≤ 5% 紧急 |
| 水浸检测 | 地面积水、管道漏水 | 实时（事件驱动） | 触发即告警 |
| 坑位占用 | 各隔间是否有人使用 | 实时（事件驱动） | 用于状态展示，非告警 |
| 温湿度 | 环境舒适度 | 1 分钟 | 超出舒适范围提醒（可选） |

```mermaid
flowchart TB
    subgraph 触发源["任务触发源"]

    T1 & T2 & T3 & T4 --> E1

    style 触发源 fill:#e8f5e9,stroke:#2e7d32
    style 调度引擎 fill:#e3f2fd,stroke:#1565c0
    style 约束条件 fill:#fff3e0,stroke:#e65100
```

| 功能 | 描述 | 业务规则 |
|------|------|---------|
| 排班管理 | 支持周/月排班，可手动调整 | 考虑休假、加班、换班 |
| 自动任务生成 | 根据传感器数据自动创建清洁任务 | 可配置触发规则（人流量/时间/环境） |
| 智能派单 | 基于距离、工作量、技能自动匹配最优人员 | 支持手动覆盖 |
| 突发响应 | 处理用户投诉、设备故障等紧急事件 | 优先级高于常规任务 |
| SLA 引擎 | 定义清洁响应时间、完成时间标准 | 超时自动升级通知 |

### 3.3 清洁执行管理

```mermaid
stateDiagram-v2
    [*] --> 待接单 : 任务创建
    待接单 --> 已接单 : 保洁员确认
    待接单 --> 已超时 : 超时未接
    已超时 --> 待接单 : 重新派单

    已接单 --> 执行中 : 扫码签到
    执行中 --> 待审核 : 完成清洁 & 拍照上传
    待审核 --> 已完成 : 质检通过
    待审核 --> 待返工 : 质检不通过
    待返工 --> 执行中 : 重新执行

    已完成 --> [*]

    note right of 执行中
        - 按 Checklist 逐项操作
        - 清洁前拍照
        - 清洁后拍照
        - 异常上报
    end note
```

| 功能 | 描述 | 关键设计 |
|------|------|---------|
| 任务接收 | 推送 + 列表查看待办任务 | 支持抢单/指派两种模式 |
| 扫码签到 | 扫描卫生间门口二维码签到 | GPS + 二维码双重校验；支持离线 |
| Checklist | 标准化清洁步骤引导 | 管理员可按卫生间类型自定义 |
| 拍照上传 | 清洁前后对比照片 | 强制拍照、EXIF 校验防篡改 |
| 耗材补充 | 记录补充的耗材种类和数量 | 关联库存消耗 |
| 故障上报 | 发现设备故障一键上报 | 自动创建维修工单 |
| 签退完成 | 确认所有步骤完成后签退 | 自动计算清洁耗时 |

### 3.6 数据分析与看板

| 看板类型 | 核心指标 | 使用角色 |
|---------|---------|---------|
| 实时大屏 | 各卫生间实时状态、人流热力图、告警信息 | 保洁主管、物业经理 |
| 运营看板 | 任务完成率、平均响应时间、SLA 达标率 | 保洁主管、物业经理 |
| 质量看板 | 质检通过率、返工率、评分分布 | 保洁主管、物业经理 |
| 满意度看板 | 评价星级分布、趋势变化、Top 投诉 | 物业经理、区域总监 |
| 成本看板 | 人力成本、耗材消耗、设备维护费 | 物业经理、管理层 |
| 对比分析 | 多场所横向对比、环比/同比趋势 | 区域总监、管理层 |

## 5. 核心业务流程

### 5.1 完整清洁作业流程

```mermaid
sequenceDiagram
    participant IoT as IoT传感器
    participant GW as 边缘网关
    participant SRV as 调度服务
    participant TL as 保洁主管
    participant CL as 保洁员APP
    participant AI as AI质检
    participant PU as 公众用户

    IoT->>GW: 上报数据（人流/环境）
    GW->>SRV: 推送至云平台
    SRV->>SRV: 触发规则引擎
    SRV->>CL: 派发清洁任务
    CL->>CL: 确认接单
    CL->>CL: 扫码签到 + 拍照（清洁前）
    CL->>CL: 按 Checklist 执行清洁
    CL->>CL: 拍照（清洁后）+ 签退
    CL->>SRV: 提交清洁记录
    SRV->>AI: 发送前后照片
    AI->>SRV: 返回质检评分
    alt 评分合格
        SRV->>TL: 通知质检通过
        SRV->>SRV: 任务标记完成
    else 评分不合格
        SRV->>TL: 通知需复核
        TL->>SRV: 确认返工
        SRV->>CL: 派发返工任务
    end
    PU->>SRV: 扫码评价
    SRV->>SRV: 关联满意度数据
```
